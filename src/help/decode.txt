Usage: {frad} decode [OPTION]... INPUT-FILE
   or: {frad} dec [OPTION]... INPUT-FILE

Decode Fourier Analogue-in-Digital format audio files to Pulse Code Modulated
raw audio data, performing inverse spectral transformation with optional
error correction and format conversion.

When INPUT-FILE is -, read standard input. The decoder automatically detects
and validates the FrAD format header, extracting all necessary parameters
for accurate reconstruction of the original audio signal.

Mandatory arguments to long options are mandatory for short options too.

Output format specification:
  -f, --format FMT              specify output PCM sample format
      --fmt FMT                 abbreviated form of --format
      --pcm FMT                 alternative form emphasizing PCM nature
                                Default: f64be (64-bit float, big-endian)

                                Integer formats:
                                  u8        - unsigned 8-bit
                                  s16le/be  - signed 16-bit little/big-endian
                                  s24le/be  - signed 24-bit little/big-endian
                                  s32le/be  - signed 32-bit little/big-endian

                                Floating-point formats:
                                  f32le/be  - 32-bit IEEE 754 single precision
                                  f64le/be  - 64-bit IEEE 754 double precision


Error correction:
  -e, --ecc                     enable error detection and correction
      --enable-ecc              descriptive form of --ecc
                                Default: disabled for speed

                                When enabled, the decoder utilizes
                                embedded Reed-Solomon parity symbols
                                to detect and correct corrupted frames.
                                This increases decoding time by
                                approximately 15-20% but ensures
                                data integrity.


Input/Output control:
  -o, --output FILE             write decoded output to FILE
      --out FILE                abbreviated form of --output
                                If omitted, output is written to stdout
                                or INPUT-FILE with appropriate extension
                                (.pcm, .raw, .f32, .f64) based on format

      --force                   overwrite existing output files
  -y                            same as --force

Informational output:
  -v, --loglevel N              set log verbosity level (0-1)
                                0: silent, 1: informational

Examples:
  Basic decoding to default format:
    {frad} decode input.frad -o output.pcm

  Decode with error correction to 16-bit WAV-compatible format:
    {frad} decode damaged.frad --ecc --format s16le --output fixed.raw

  Pipe decoding for real-time processing:
    {frad} decode --format f32le - < input.frad | \
    sox -t raw -r 44100 -c 2 -e floating-point -b 32 - output.wav


Exit status:
  0  if successful
  1  if errors occurred
  2  if file format is invalid

If the info and {frad} programs are properly installed at your site, the
command `info frad decode' should provide complete documentation.
