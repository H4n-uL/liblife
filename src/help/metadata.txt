Usage: {frad} meta META-ACTION [OPTION]... INPUT-FILE
   or: {frad} metadata META-ACTION [OPTION]... INPUT-FILE

Manipulate extensible metadata chunks embedded within
Fourier Analogue-in-Digital format files, supporting multiple encoding schemes
and hierarchical data structures.

META-ACTION specifies the operation to perform on the metadata chunk.
Valid actions: add, remove, rm-img, overwrite, parse

Mandatory arguments to long options are mandatory for short options too.

Metadata actions (mutually exclusive):
  add         Append new metadata entries to existing metadata chunk
              Preserves existing entries while adding new key-value pairs
              Duplicate keys are permitted and accumulate

  remove      Selectively remove metadata entries by key
              Removes all entries matching the specified key
              Preserves all other metadata and embedded images

  rm-img      Remove embedded cover artwork image
              Preserves all text metadata entries
              This operation is irreversible

  overwrite   Replace entire metadata chunk with new data
              WARNING: Destroys all existing metadata
              Use with caution on files with existing metadata

  parse       Extract and export metadata to structured format
              Outputs JSON representation of all metadata
              Includes Base64-encoded binary attachments

Action-specific parameters:

For actions 'add' and 'overwrite':
  -m, --meta KEY VALUE          add metadata key-value pair
      --tag KEY VALUE           same as --meta
                                Multiple invocations accumulate entries
                                Keys are case-sensitive UTF-8 strings
                                Values support full Unicode range

      --jsonmeta FILE           import metadata from JSON format file
      --jm FILE                 abbreviated form of --jsonmeta
                                See '{frad} jsonmeta' for format specification
                                Supports structured and binary metadata

      --vorbis-meta FILE        import Vorbis comment format metadata
      --vm FILE                 abbreviated form of --vorbis-meta
                                See '{frad} vorbismeta' for format details
                                Simple key=value text format

      --image FILE              embed album artwork or cover image
      --img FILE                abbreviated form of --image
                                Supported formats: JPEG, PNG, GIF, BMP, WebP
                                Maximum size: 16 MB (recommended: < 1 MB)
                                Replaces existing image if present

For action 'remove':
  -m, --meta KEY                specify metadata key to remove
      --tag KEY                 alternative form
                                Removes ALL entries with this key
                                Case-sensitive matching
                                Use quotes for keys with spaces

For action 'rm-img':
  No additional parameters required.
  Removes the embedded image while preserving all text metadata.

For action 'parse':
  -o, --output FILE             write extracted metadata to FILE
      --out FILE                abbreviated form of --output
                                Default: write to standard output
                                Format: JSON with proper indentation


Global options:
      --force                   overwrite without confirmation
  -y                            same as --force

Metadata key naming conventions:
  Standard fields (recommended for compatibility):
    TITLE         - Track or work title
    ARTIST        - Primary artist or performer
    ALBUM         - Album or collection name
    DATE          - Release date (ISO 8601 format preferred)
    GENRE         - Musical genre classification
    COMMENT       - Free-form annotations
    COPYRIGHT     - Copyright notice
    LICENSE       - Distribution license
    TRACKNUMBER   - Track sequence number
    DISCNUMBER    - Disc or volume number
    ALBUMARTIST   - Album's primary artist
    COMPOSER      - Original composer
    PERFORMER     - Performance credits
    CONDUCTOR     - Orchestra conductor
    ORGANIZATION  - Label or publisher
    ISRC          - International Standard Recording Code
    ENCODER       - Encoding software and settings

  Custom fields:
    Any UTF-8 string not starting with underscore
    Convention: Use UPPERCASE for consistency
    Examples: MOOD, BPM, KEY, RATING, LYRICS

  Reserved fields:
    _* (underscore prefix) - System use only
    Do not create or modify these fields

Examples:
  Add metadata to existing file:
    {frad} meta add song.frad --meta ARTIST Bach --meta YEAR 1721

  Import metadata from JSON and add cover:
    {frad} meta add album.frad --jsonmeta tags.json --image cover.jpg

  Replace all metadata:
    {frad} meta overwrite file.frad --vm new_tags.txt

  Extract metadata to file:
    {frad} meta parse collection.frad --output metadata.json

  Remove specific metadata key:
    {frad} meta remove audio.frad --meta COMMENT

  Remove cover image only:
    {frad} meta rm-img podcast.frad

  Batch processing:
    for f in *.frad; do
      {frad} meta add "$f" --meta PROCESSED "$(date)"
    done

Exit status:
  0  if operation completed successfully
  1  if errors occurred
  2  if action or parameters are invalid

The full documentation for {frad} metadata is maintained as a Texinfo manual.
If the info and {frad} programs are properly installed at your site, the
command `info frad metadata' should provide complete documentation.
