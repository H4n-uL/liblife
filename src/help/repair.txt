Usage: {frad} repair [OPTION]... INPUT-FILE
   or: {frad} ecc [OPTION]... INPUT-FILE

Apply Reed-Solomon forward error correction algorithms to detect and repair
corrupted frames in Fourier Analogue-in-Digital format files, utilizing
embedded parity symbols to reconstruct damaged audio data.

When INPUT-FILE is -, read standard input. The repair process analyzes each
frame independently, identifying corruption through syndrome calculation and
applying algebraic error correction when possible.

Mandatory arguments to long options are mandatory for short options too.

Error correction parameters:
  -e, --ecc DATA,CODE           specify Reed-Solomon code parameters
      --enable-ecc DATA CODE    same as --ecc
                                Default: 96,24 (20% redundancy)
                                Valid ranges: DATA: 1-255, CODE: 0-(255-DATA)

                                The error correction capability is:
                                - Random errors: up to CODE/2 symbols per block
                                - Burst errors: up to CODE consecutive symbols
                                - Erasures: up to CODE known error positions

                                Common configurations:
                                  96 24  - Standard (12.5% overhead)
                                  192 32 - Maximum protection (14.3% overhead)
                                  223 32 - Highest data rate (12.5% overhead)

Input/Output control:
  -o, --output FILE             write repaired file to FILE
      --out FILE                abbreviated form of --output
                                If omitted, creates INPUT-FILE.repaired.frad


      --force                   overwrite existing output files
  -y                            same as --force


Informational output:
  -v, --loglevel N              set log verbosity level (0-1)
                                0: silent, 1: show repair statistics

Examples:
  Basic repair with default parameters:
    {frad} repair damaged.frad -o repaired.frad

  Maximum error correction:
    {frad} repair corrupted.frad --ecc 192 32 -o fixed.frad

  Batch repair with high protection:
    for f in *.frad; do
      {frad} repair "$f" --ecc 223 32 -o "repaired_$f"
    done


Error correction theory:
  The Reed-Solomon implementation uses Galois Field GF(2^8) arithmetic
  with generator polynomial construction for systematic encoding. Error
  location is performed via Berlekamp-Massey algorithm with Chien search,
  while error magnitude uses Forney's algorithm.

  Success probability for random bit errors:

  where n = block size, k = correction capability, p = bit error rate

Exit status:
  0  if all errors were corrected successfully
  1  if some frames remain corrupted after repair
  2  if file is irreparably damaged
  3  if input file format is invalid
